Ontwikkeling

Waarom kies je voor de opleiding programmeren?

Ik koos voor Graduaat Programmeren omdat ik graag problemen oplos en het leukste leer door dingen echt te bouwen. Programmeren combineert logisch denken met creativiteit: je start met een idee en eindigt met iets dat effectief werkt. Dit jaar merkte ik dat ik veel voldoening haal uit het stap voor stap verbeteren van code en het overzichtelijk maken van een projectstructuur. Daarnaast wil ik een concrete basis opbouwen die aansluit bij het werkveld, zoals versiebeheer met Git, webtechnologie (HTML/CSS) en C#. Mijn doel is om mijn fundament sterk te maken zodat ik later professioneel kan bijdragen in een team, duidelijke afspraken kan volgen en grotere projecten kan aanpakken zonder het overzicht te verliezen.

Waarom denk je dat dit beroep bij jou past?

Ik denk dat het beroep developer bij mij past omdat ik graag bijleer en niet opgeef wanneer iets niet meteen lukt. Tijdens opdrachten merkte ik dat ik het fijn vind om fouten op te sporen en oplossingen te testen tot het stabiel werkt. Ik werk ook graag gestructureerd: kleine stappen, duidelijke commits en een overzichtelijke folder-structuur. Daarnaast kan ik feedback gebruiken om mijn werk te verbeteren, bijvoorbeeld door mijn code leesbaarder te maken of een pagina beter responsive te maken. In een team wil ik helder communiceren over wat ik doe en wat ik nodig heb, zodat het werk vlot vooruitgaat en iedereen dezelfde verwachtingen heeft.

Vijf rollen (OLR)

Rol: Ontwerper

Waar sta ik?

Ik kan een structuur uittekenen (sections/cards/tables) en consistent houden, maar ik wil sneller betere keuzes maken zonder te veel te herwerken.

Wat lukt al / nog niet?

Lukt al: layout plannen, componenten hergebruiken, consistent thema.

Nog niet: alles meteen “goed” ontwerpen zonder iteratie.

3 voorbeelden

  1. Voorbeeld 1: Hero + sections opgebouwd met grid/flex.
  2. Voorbeeld 2: Navigatie omgezet naar dropdown op small widths.
  3. Voorbeeld 3: Consistente cards/tables styling toegepast.

Rol: Programmeur

Waar sta ik?

Ik kan opdrachten zelfstandig uitwerken in HTML/CSS en C# (basis) en ik leer beter structureren, debuggen en hergebruiken.

Wat lukt al / nog niet?

Lukt al: basis JS, C# console/WPF basics, Git workflow.

Nog niet: grote projecten meteen schaalbaar opzetten zonder refactor.

3 voorbeelden

  1. Voorbeeld 1: CV/portfolio website met herbruikbare CSS-classes.
  2. Voorbeeld 2: C# console opdracht met inputvalidatie en methodes.
  3. Voorbeeld 3: Branches gebruikt + duidelijke commit messages.

Rol: Tester

Waar sta ik? Ik test mijn werk met scenario’s (desktop/mobiel, verschillende schermgroottes) en controleer console errors.

Lukt al: regressietesten na wijzigingen, bugs reproduceren en fixen.

Nog niet: formeel testplan schrijven voor grotere opdrachten.

3 voorbeelden

  1. Voorbeeld 1: Responsive check op 3 breakpoints.
  2. Voorbeeld 2: Anchor links getest (nav naar juiste section).
  3. Voorbeeld 3: Debugging in C#: breakpoints + output logging.

Rol: Communicator / Teamspeler

Waar sta ik? Ik kan feedback vragen en verwerken, en documenteer kort wat ik aangepast heb.

Lukt al: duidelijke status (“wat gedaan / wat blokkeert”), notities bij logboek.

Nog niet: altijd proactief communiceren vóór iets misloopt.

3 voorbeelden

  1. Voorbeeld 1: Feedback verwerkt in layout en navigatie.
  2. Voorbeeld 2: Logboek per week bijgehouden met kernpunten.
  3. Voorbeeld 3: Reflecties geschreven met concrete voorbeelden.

Rol: Levenslang lerende IT-professional

Waar sta ik? Ik kan zelfstandig bijleren via documentatie en pas dat toe in opdrachten.

Lukt al: nieuwe CSS/HTML technieken proberen en iteratief verbeteren.

Nog niet: altijd tijd reserveren voor “extra” leren naast deadlines.

3 voorbeelden

  1. Voorbeeld 1: CSS-only dropdown via <details> toegepast.
  2. Voorbeeld 2: Minder JS gebruikt: HTML/CSS-first oplossingen.
  3. Voorbeeld 3: Git best practices (kleine commits, duidelijke messages).

Reflectie — AIM-model

Intro

Tijdens Werkplekleren 1 werkte ik aan opdrachten rond Scratch, Git/GitHub, web (HTML/CSS), C# en een beetje JavaScript. In dit portfolio toon ik niet enkel het eindresultaat, maar vooral wat ik geleerd heb tijdens het proces en hoe ik gegroeid ben.

Bijzonder positief element

Ik vond het positief dat ik doorheen het semester duidelijker structuur begon te gebruiken: kleinere taken, duidelijke commits en een consistente layout in mijn websites. Daardoor kon ik fouten sneller terugvinden en verbeteringen makkelijker toevoegen.

Minder leuk element

Soms liep ik vast door kleine fouten (bv. CSS die niet toegepast werd of een link die niet werkte), waardoor ik tijd verloor. Dat was frustrerend, vooral wanneer ik dacht dat alles correct was. Ik merkte ook dat ik soms te lang bleef doorgaan zonder pauze.

Nieuwe inzichten

Ik heb geleerd dat “kleine stappen + testen” het verschil maakt. Als ik na elke wijziging meteen test (desktop en mobiel), vind ik sneller waar iets fout loopt. Ook heb ik gemerkt dat eenvoudige oplossingen (HTML/CSS-first, minimale JS) vaak het best werken.

Persoonlijk succes

Een persoonlijk succes was dat ik mijn websites een consistente stijl kon geven en tegelijk responsive kon houden. Ik heb ook geleerd om feedback om te zetten in concrete verbeterpunten, in plaats van het enkel te lezen en te vergeten. Dit geeft me vertrouwen om in volgende projecten zelfstandiger te werken.

Kernkwadrant(en)

Kernkwadrant

Kernkwaliteit: Doorzettingsvermogen

Ik blijf zoeken tot iets werkt, ook wanneer ik vastloop op bugs of styling.

Valkuil: Te lang blijven hangen in details

Ik kan soms te veel tijd verliezen aan “perfect” maken.

Uitdaging: Focus op “done” i.p.v. “perfect”

Eerst werkend en duidelijk, daarna pas verfijnen.

Allergie: Onzorgvuldigheid

Slordige code/structuur zorgt voor extra fouten en frustratie.

Motivatie

Mijn kernkwaliteit helpt mij om opdrachten af te krijgen, vooral wanneer ik vastloop op bugs of details in de layout. Mijn valkuil is dat ik soms te lang blijf verfijnen terwijl de basis al af is. Daarom is mijn uitdaging om eerst een minimum werkende versie te maken en pas daarna te verbeteren. Ik ben aan het werken aan het plannen en inschatten van hoelang ik werk aan een taak.

X-factor

Ondernemend & innovatief

  • Ik zoek eenvoudige oplossingen die onderhoudbaar zijn (CSS-first).
  • Ik experimenteer met UI en maak het consistent doorheen pages.
  • Ik verbeter iteratief met kleine wijzigingen + testen.

Samenwerken & communiceren

  • Ik vraag feedback en verwerk die in concrete acties.
  • Ik documenteer wat ik aangepast heb (logboek/notes).
  • Ik maak duidelijke commits zodat wijzigingen te volgen zijn.

Levenslang leren

  • Ik leer nieuwe technieken wanneer nodig (dropdown, responsive layout).
  • Ik probeer dingen zelfstandig uit en pas toe in opdrachten.
  • Ik gebruik fouten als feedback om mijn aanpak te verbeteren.

Duurzaam & bewust (SDG)

  • Ik maak herbruikbare componenten en vermijd overbodige code.
  • Ik houd structuur en naming consistent voor onderhoudbaarheid.
  • Ik test op verschillende schermen om herwerk te beperken.
